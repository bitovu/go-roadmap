# Циклы в Go

---

## Краткий ответ на собеседовании

> В Go существует только один цикл — `for`.  
> Он универсальный и используется как обычный цикл, как `while`,
> а также для итерации по коллекциям через `range`.  
>  
> Классических `while` и `do-while` в Go нет.

---

## База

### Классический `for`

```go
for i := 0; i < 10; i++ {
    fmt.Println(i)
}
```

* состоит из инициализации, условия и шага
* полностью аналогичен for в C-подобных языка

  ---

### `for` как `while`

В Go нет отдельного цикла `while`.  
Его роль выполняет `for` с одним условием.

```go
i := 0
for i < 10 {
    fmt.Println(i)
    i++
}
```
* условие проверяется перед каждой итерацией
* используется, когда шаг цикла нефиксированный

--- 

## Бесконечный цикл
```
for {
    // бесконечный цикл
}
```
* используется в серверах, воркерах, обработчиках событий
* выход осуществляется через break или return
---

## Цикл `for range` 

`range` используется для перебора коллекций и других структур данных.

---

### Перебор слайса

```go
nums := []int{1, 2, 3}

for i, v := range nums {
    fmt.Println(i, v)
}
```
* i — индекс элемента
* v — значение
  
Если индекс не нужен:
```
for _, v := range nums {
    fmt.Println(v)
}
```

---
### Перебор строки
```
s := "Go语言"

for i, r := range s {
    fmt.Println(i, r)
}
```
* перебор выполняется по рунам, а не по байтам
* i — индекс байта
* r — rune (Unicode-символ)

---

### Перебор `map`
```
m := map[string]int{
    "a": 1,
    "b": 2,
}

for k, v := range m {
    fmt.Println(k, v)
}
```
* порядок обхода map не гарантирован
